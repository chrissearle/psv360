## Project Purpose

Self-hosted 360┬░ panorama viewer. A Nuxt 4 app backed by Photo Sphere Viewer's Virtual Tour plugin and the Equirectangular Tiles adapter. Reads a `PANO_DIR` directory of equirectangular images and a `config.json` defining scenes and hotspot links between them.

## Commands

```bash
pnpm install      # install deps
pnpm dev          # dev server (default: http://localhost:3000)
pnpm build        # production build
pnpm preview      # preview production build
pnpm lint         # ESLint
pnpm lint:fix     # ESLint with auto-fix
pnpm typecheck    # vue-tsc type check

node --experimental-strip-types scripts/generate-config.ts   # scan PANO_DIR, add new scenes, generate tiles
```

## Tech Stack

- **Nuxt 4** ŌĆö app in `app/`, server in `server/`, shared types in `shared/types/`
- **Nuxt UI** ŌĆö component library; default theme is overridden in `assets/css/main.css`. Use `UCard`, `UAlert`, `UIcon`, `UApp` etc.
- **Photo Sphere Viewer** ŌĆö client-side only. Uses `EquirectangularTilesAdapter` for progressive tile loading and `VirtualTourPlugin` for hotspot navigation. The viewer component is `PanoViewer.client.vue` (`.client.` suffix means Nuxt skips SSR for it automatically). Always import PSV and its CSS inside `.client.vue` files or behind `<ClientOnly>`. The tiles adapter has no `index.css` ŌĆö do not import one.
- **pnpm** ŌĆö exact versions only (no `^` or `~`); Renovate manages update PRs.

## Architecture

### Data flow

1. `PANO_DIR` env var (default `./data`) points to a directory with images + `config.json`
2. `server/utils/panoConfig.ts` reads and in-memory caches the config
3. Server API routes expose scenes and stream image files:
   - `GET /api/scenes` ŌĆö all scenes
   - `GET /api/scenes/[id]` ŌĆö single scene
   - `GET /api/image/[...path]` ŌĆö serves any file from `PANO_DIR` with path-traversal protection
4. Front-end pages fetch from these routes via `useFetch`

### Routing

- `/` ŌĆö scene gallery (`app/pages/index.vue`)
- `/pano/[id]` ŌĆö full-screen PSV viewer (`app/pages/pano/[id].vue`)
- `/embed/[id]` ŌĆö frameless embed page with static thumbnail + link, `layout: false` (`app/pages/embed/[id].vue`)

### Directory layout per scene

Each scene lives under `PANO_DIR/YYYY/MM/DD/<name>/` and contains:

```
panorama.jpg      # full-resolution equirectangular source
thumb.jpg         # thumbnail used in gallery and embed pages
base.jpg          # low-res base image shown by PSV while tiles load (generated)
tiles/
  tile_COL_ROW.jpg  # 32├Ś16 grid of tiles (generated)
```

`base.jpg` and `tiles/` are generated by `scripts/generate-config.ts` ŌĆö never edit them manually.

### Config format

```json
{
  "scenes": [
    {
      "id": "2022-10-02-slug",
      "name": "Display Name",
      "path": "2022/10/02/slug",
      "date": "2022-10-02",
      "tiles": { "width": 17920, "height": 8960, "cols": 32, "rows": 16 },
      "hotspots": [
        {
          "id": "h1",
          "targetId": "other-slug",
          "yaw": 1.2,
          "pitch": 0.0,
          "label": "optional"
        }
      ],
      "defaultPosition": { "yaw": 0.0, "pitch": 0.0 }
    }
  ]
}
```

- `path` is relative to `PANO_DIR`; the app derives `panorama.jpg`, `thumb.jpg`, `base.jpg`, and `tiles/tile_COL_ROW.jpg` from it automatically.
- `tiles` is written by `generate-config.ts` after slicing. PSV requires `cols` and `rows` to both be powers of 2. Current fixed values: `cols=32, rows=16`.
- `id` is derived from the directory path with `/` replaced by `-`.
- Scenes without `tiles` fall back to loading `panorama.jpg` directly.

Types live in `shared/types/index.ts` and are usable in both `app/` and `server/` via `~~/shared/types`.

### Tile generation

`scripts/generate-config.ts` (run with `node --experimental-strip-types`):

- Scans `PANO_DIR` for directories containing `panorama.jpg`
- Adds new scenes to `config.json`
- Backfills missing `date` fields on existing scenes
- Generates `base.jpg` and `tiles/` for any scene missing them, then writes `tiles` metadata to `config.json`
- Skips a scene's tile generation if `tiles/` already exists
- Requires ImageMagick (`magick`) on `PATH`

### Styling

Dark theme by default. `assets/css/main.css` sets CSS variables on `:root` and imports Tailwind + Nuxt UI. Do not use Nuxt UI's default color palette ŌĆö stay in `neutral-*` tones with `primary` for interactive accents.

## Hosting

The site may run at 360.chrissearle.org but this can change. Nothing in the app may reference the hostname ŌĆö no hardcoded URLs, no canonical link meta, no domain-specific config.

## Deployment requirements

- Mount image volume into container
- Set `PANO_DIR` to the mount path
- Flux manifests live in a separate repo; do not add k8s/flux config here
